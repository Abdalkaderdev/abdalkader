## What is the Abdalkader API?

The Abdalkader API is a RESTful API that provides programmatic access to:

- **Health Monitoring**: Check system status and health metrics
- **Performance Tracking**: Submit and analyze performance metrics
- **Error Reporting**: Report and track errors for debugging
- **Analytics**: Access analytics and usage data

## Base URLs

The API is available at different base URLs depending on your environment:

| Environment | Base URL | Description |
|------------|----------|-------------|
| **Production** | `https://abdalkader.dev/api` | Live production API |
| **Staging** | `https://staging.abdalkader.dev/api` | Staging environment for testing |
| **Local** | `http://localhost:3000/api` | Local development server |

## API Versioning

The current API version is **v1.0.0**. All endpoints are versioned and backward compatible within the same major version.

```http
GET /api/v1/health
```

## Response Format

All API responses are returned in JSON format with the following structure:

### Success Response

```json
{
  "success": true,
  "data": { ... },
  "message": "Optional success message"
}
```

### Error Response

```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error message",
    "details": { ... }
  }
}
```

## HTTP Status Codes

The API uses standard HTTP status codes:

| Code | Meaning | Description |
|------|---------|-------------|
| `200` | OK | Request succeeded |
| `201` | Created | Resource created successfully |
| `400` | Bad Request | Invalid request parameters |
| `401` | Unauthorized | Authentication required or failed |
| `403` | Forbidden | Insufficient permissions |
| `404` | Not Found | Resource not found |
| `405` | Method Not Allowed | HTTP method not supported |
| `429` | Too Many Requests | Rate limit exceeded |
| `500` | Internal Server Error | Server error |
| `503` | Service Unavailable | Service temporarily unavailable |

## SDKs and Libraries

While we don't provide official SDKs yet, our API is designed to work with any HTTP client. Popular options include:

- **JavaScript**: `fetch`, `axios`, `node-fetch`
- **Python**: `requests`, `httpx`
- **Go**: `net/http`
- **Ruby**: `net/http`, `httparty`
