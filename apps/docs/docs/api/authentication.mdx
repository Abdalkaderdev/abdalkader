# Authentication

The Abdalkader API uses API key authentication for most endpoints. This guide explains how to authenticate your requests.

## Authentication Methods

The API supports two authentication methods:

1. **API Key** (Recommended) - Simple header-based authentication
2. **Bearer Token** (JWT) - Token-based authentication for advanced use cases

## API Key Authentication

API keys are the simplest and most common way to authenticate. Get your API key from the [API Dashboard](https://abdalkader.dev/api/dashboard).

### How It Works

Include your API key in the `X-API-Key` header with every request:

```http
GET /api/health HTTP/1.1
Host: abdalkader.dev
X-API-Key: your-api-key-here
```

## Bearer Token Authentication

For advanced use cases, you can use JWT bearer tokens. Include the token in the `Authorization` header:

```http
GET /api/health HTTP/1.1
Host: abdalkader.dev
Authorization: Bearer your-jwt-token-here
```

## Getting Your API Key

1. Visit the [API Dashboard](https://abdalkader.dev/api/dashboard)
2. Sign in or create an account
3. Navigate to "API Keys"
4. Click "Create New API Key"
5. Copy and securely store your API key

<Warning>
  **Never expose your API key in client-side code or public repositories.** API keys should be kept secret and only used in server-side applications or secure environments.
</Warning>

## API Key Best Practices

### ✅ Do

- Store API keys in environment variables
- Use different keys for different environments (dev, staging, production)
- Rotate keys regularly
- Use the minimum permissions required
- Monitor key usage

### ❌ Don't

- Commit API keys to version control
- Share keys publicly or in client-side code
- Use the same key for multiple applications
- Leave unused keys active

## Authentication Errors

If authentication fails, you'll receive a `401 Unauthorized` response:

```json
{
  "success": false,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Invalid or missing API key"
  }
}
```

### Common Issues

| Error | Cause | Solution |
|-------|-------|----------|
| `UNAUTHORIZED` | Missing or invalid API key | Check that the `X-API-Key` header is included and correct |
| `INVALID_KEY` | API key format is incorrect | Verify the key matches the format from the dashboard |
| `EXPIRED_KEY` | API key has expired | Generate a new key from the dashboard |
| `REVOKED_KEY` | API key has been revoked | Contact support or generate a new key |

## Reusable Client Examples

### JavaScript Client Class

```javascript
class AbdalkaderAPI {
  constructor(apiKey, baseURL = 'https://abdalkader.dev/api') {
    this.apiKey = apiKey;
    this.baseURL = baseURL;
  }

  async request(endpoint, options = {}) {
    const url = `${this.baseURL}${endpoint}`;
    const response = await fetch(url, {
      ...options,
      headers: {
        'Content-Type': 'application/json',
        'X-API-Key': this.apiKey,
        ...options.headers
      }
    });

    if (!response.ok) {
      const error = await response.json();
      throw new Error(error.message || 'Request failed');
    }

    return response.json();
  }

  async getHealth() {
    return this.request('/health');
  }

  async submitPerformanceMetric(metric) {
    return this.request('/performance', {
      method: 'POST',
      body: JSON.stringify(metric)
    });
  }

  async reportError(error) {
    return this.request('/errors', {
      method: 'POST',
      body: JSON.stringify(error)
    });
  }
}

// Usage
const api = new AbdalkaderAPI(process.env.API_KEY);
const health = await api.getHealth();
```

### Python Client Class

```python
import requests
from typing import Dict, Any, Optional

class AbdalkaderAPI:
    def __init__(self, api_key: str, base_url: str = 'https://abdalkader.dev/api'):
        self.api_key = api_key
        self.base_url = base_url
        self.headers = {
            'Content-Type': 'application/json',
            'X-API-Key': api_key
        }

    def request(self, endpoint: str, method: str = 'GET', data: Optional[Dict] = None) -> Dict[str, Any]:
        url = f'{self.base_url}{endpoint}'
        response = requests.request(
            method=method,
            url=url,
            headers=self.headers,
            json=data
        )
        response.raise_for_status()
        return response.json()

    def get_health(self) -> Dict[str, Any]:
        return self.request('/health')

    def submit_performance_metric(self, metric: Dict[str, Any]) -> Dict[str, Any]:
        return self.request('/performance', method='POST', data=metric)

    def report_error(self, error: Dict[str, Any]) -> Dict[str, Any]:
        return self.request('/errors', method='POST', data=error)

# Usage
api = AbdalkaderAPI(os.getenv('API_KEY'))
health = api.get_health()
```

## Testing Authentication

You can test your authentication setup using the health endpoint (which doesn't require auth) or by making a simple authenticated request:

```bash
# Test without authentication
curl https://abdalkader.dev/api/health

# Test with API key
curl -H "X-API-Key: your-api-key-here" \
     https://abdalkader.dev/api/health
```
